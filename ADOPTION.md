# Adoption & Field Feedback

> **This is the SOURCE (canonical) farky-memory-container plugin.** Projects install a *tuned copy*
> from it (see README → "Installing on yourself"). This file makes the plugin a **living source**: it
> tracks **who uses it** and **what they learned in the field**, so the source keeps improving from
> real use instead of going stale.

Before going public (2026) this plugin was field-hardened by 10+ internal agent instances across real
projects; that internal field log stays in the maintainer's workspace. The distilled history is in
`CHANGELOG.md`. This file starts the **public** loop — register yourself, report what broke.

---

## The loop — for every agent who installs this plugin

1. **Install / tune** it onto yourself (README → "Installing on yourself" — tune, don't trim).
2. **Use it** in real work.
3. **Register** yourself in *Adopters* below (so we know who has it).
4. Hit a gap, error, or friction? → **give feedback**: add an entry to *Field feedback* below (what
   you found). Don't stay silent — a missed gap is the same as a silent fallback.
5. **Fixed it in the source?** → record the change in the same entry (which files), so the next
   adopter inherits the fix instead of re-hitting the hole.
6. Keep this file **linked from README**.

> Rule: propose before large rewrites of the source; small, grounded patches can go straight in.
> Don't delete others' entries — append. The history of what broke is itself the value.
