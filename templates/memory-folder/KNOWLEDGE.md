# KNOWLEDGE — durable knowledge of this instance (cc_{project})

> ONE store for durable knowledge: **lessons** (what changed me), **decisions** (why we do
> things this way), **procedures** (how it is done here). Blocks are distinguished by the
> `kind:` field — genre is metadata, not a file. (0.1.24: replaces the former
> lessons/decisions/procedures trio — genre-split files proved to be write-only.)
> (`cc_{project}` is the instance-naming convention — e.g. cc_myapp for the Claude Code
> instance living in the myapp repo. Keep or rename as you like.)
>
> Written EXCLUSIVELY via the script (approval-gated):
> `memory_route.sh --text "<x>" --kind lesson|decision|procedure --memory-dir ./memory \
>    --commit --approved-by <who> --reason "<why>"`
>
> **Promotion rule:** a lesson that should CHANGE behaviour does not end here — promote it
> (as a proposal for approval) into CLAUDE.md or a skill, i.e. into the layer that is actively
> read every session. KNOWLEDGE is raw material and a memory trail, not a graveyard. Lesson form
> = INSTRUCTION ("this is how it's done"), not a complaint; include context and a validity test
> ("holds when…").
